## Design Context

**Product**: Document sharing service for AI-generated HTML/Markdown content
**URL**: share.eregistrations.dev

**Target audience**: UNCTAD staff using AI agents (Claude Code, Codex) to publish documents programmatically. Rarely manual uploads via web UI. Humans primarily consume (view) the shared content.

**Use cases**:
- AI agents publish analysis reports, HTML artifacts, markdown docs via API
- Staff share permanent links to generated content with colleagues and stakeholders
- Occasional manual upload of documents through the web form

**Brand personality**: Professional tool. Clean, trustworthy, institutional. Should feel like a serious utility — not playful, not developer-grungy.

**Brand relationship**: Standalone identity. The `.eregistrations.dev` domain provides enough connection to the ecosystem. Own visual language.

**Design principles**:
- Content is the star — the chrome should disappear when viewing documents
- Utility-first — every element earns its place
- Quiet confidence — professional without being corporate-sterile
